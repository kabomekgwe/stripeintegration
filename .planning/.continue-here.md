<context>
The PPP (Purchasing Power Parity) feature adjusts prices based on user's country to account for differences in purchasing power. This is a common practice for SaaS products selling globally.

Key implementation approach:
1. Create pricing service with tier-based multipliers
2. Apply discount at payment intent creation time
3. Store both original and adjusted amounts for analytics
4. Show transparent pricing to users with savings breakdown

The User model already has a `country` field (ISO 3166-1 alpha-2) that can be used. Currency detection is already implemented in the currency service.

</context>
